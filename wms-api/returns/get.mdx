---
title: "Get All Returns"
api: "GET /wms/returns"
description: "Get all returns"
---

### Headers
<ParamField header="x-trackstar-api-key" type="string" required>
  Your organization-level Trackstar API Key.
</ParamField>

<ParamField header="x-trackstar-access-token" type="string" required>
  Your user's access token for a specific integration (ShipHero, Extensiv, etc).
</ParamField>

### Query Params
<ParamField query="page_token" type="string" optional>
  The Trackstar API returns 250 items by default.
  If you make a query that returns more than 250 items, the API will return a `next_token` that you can pass in to get the next page of results.
  Leave this field out to get the first page.
</ParamField>

### Response

<ResponseField name="data" type="list of returns">

A list of returns. See the [Return definition](/wms-api/returns/info) for more details.

<Expandable title="Return">

<ResponseField name="id" type="string">
The unique ID of a return.
</ResponseField>

<ResponseField name="created_date" type="string">
The date the return was requested. Returned in YYYY-MM-DDTHH:MM:SSZ format.
</ResponseField>

<ResponseField name="status" type="string">
Status of the return.
</ResponseField>

<ResponseField name="line_items" type="list of line items">
List of items included in the return.
<Expandable title="Line Item" defaultOpen="true">

<ResponseField name="inventory_item_id" type="string">
Can be passed in to the `/inventory/{inventory_item_id}` endpoint for more details.
</ResponseField>

<ResponseField name="quantity" type="int">
</ResponseField>

<ResponseField name="is_restocked" type="bool or null">
Whether or not the inventory item has been restocked.
</ResponseField>

<ResponseField name="warehouse_id" type="string">
The ID of the warehouse receiving the return.
</ResponseField>

<ResponseField name="returned_date" type="string or null">
If the item has been returned, the date that it got back to the warehouse. Returned in YYYY-MM-DDTHH:MM:SSZ format.
</ResponseField>

</Expandable>
</ResponseField>


</Expandable>
</ResponseField>
