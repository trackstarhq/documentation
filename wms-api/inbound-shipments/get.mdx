<Snippet file="headers.mdx" />

### Query Params
<ParamField query="page_token" type="string" optional>
  The Trackstar API returns 250 items by default.
  If you make a query that returns more than 250 items, the API will return a `next_token` that you can pass in to get the next page of results.
  Leave this field out to get the first page.
</ParamField>

### Response

<ResponseField name="data" type="list of inbound shipments">

A list of inbound shipments. See the [Inbound Shipment definition](/wms-api/inbound-shipments/info) for more details.

<Expandable title="Inbound Shipment">

<ResponseField name="id" type="string">
The unique ID of a shipment.
</ResponseField>

<ResponseField name="created_date" type="string">
The date the shipment record was created. Returned in YYYY-MM-DDTHH:MM:SSZ format.
</ResponseField>

<ResponseField name="purchase_order_number" type="string">
The number of the purchase order associated with the inbound shipment. In some cases, will be the same as the shipment ID.
</ResponseField>

<ResponseField name="status" type="string">
The status of the inbound shipment.
</ResponseField>

<ResponseField name="supplier" type="string or null">
The supplier from which the shipment is being sent.
</ResponseField>

<ResponseField name="expected_arrival_date" type="string or null">
Date that the shipment is expected to reach the warehouse. Returned in YYYY-MM-DDTHH:MM:SSZ format.
</ResponseField>

<ResponseField name="warehouse_id" type="string or null">
ID of the warehouse that will be receiving the shipment.
</ResponseField>

<ResponseField name="line_items" type="list of line items">
List of items included in the shipment.
<Expandable title="Line Item" defaultOpen="true">

<ResponseField name="inventory_item_id" type="string">
Can be passed into the `/inventory/{inventory_item_id}` endpoint for more details.
</ResponseField>

<ResponseField name="sku" type="string or null">
</ResponseField>

<ResponseField name="quantity" type="integer">
</ResponseField>

</Expandable>
</ResponseField>

</Expandable>

</ResponseField>
